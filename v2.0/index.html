유통사 전용몰 구축 최종 기획안

(v2.0)

사용자 유형과 접속 주소에 따라 개인화된 UI(상품, 로고, 메뉴)를 제공하는 **동적 UI 시스템** 구축을 위한 최종 통합 기획안입니다.

1. 기획 요약 (Executive Summary)

프로젝트 목표: 각 유통사에게 고유 서브도메인(`[유통사코드].orderz.kr`)을 자동으로 제공하고, 접속자에 따라 맞춤형 UI를 노출하여 독립적인 전용몰 경험을 제공합니다.

핵심 기능:
  • **동적 상품 노출:** 접속한 URL과 로그인한 사용자 정보에 따라 각기 다른 상품 목록을 보여줍니다.
  • **UI 브랜딩:** 유통사별 전용 로고와 GNB 메뉴를 설정하고 노출합니다.

개발 전략: 가장 핵심적인 **'동적 상품 노출' 기능을 MVP(Minimum Viable Product)로 우선 개발**한 후, UI 브랜딩 기능을 점진적으로 확장합니다.

2. Phase 1: MVP 개발 - 동적 상품 노출

시스템의 기반을 다지고, 가장 핵심적인 '상황에 맞는 상품 노출' 기능을 구현하여 빠르게 가치를 검증합니다.

STEP 1: 인프라 설정 (Infra Setup)

와일드카드 DNS 설정: 도메인 관리 시스템에서 `*.orderz.kr` 레코드가 서버 IP를 가리키도록 설정합니다.

STEP 2: 백엔드 개발 (Back-end)

DB 스키마 확장:
  • `Products` 테이블: **'유통사 코드'** 컬럼 추가 (공용 상품은 null)
  • `Users` 테이블: **'유통사 코드'** 컬럼 추가 (소속 없는 회원은 null)
맞춤형 상품 API 개발: 아래 프로세스 흐름도에 따라 상품 목록을 결정하는 API를 구현합니다.

상품 목록 API 호출 로직 구현
API 응답 데이터를 화면에 표시하는 UI 컴포넌트 개발

3. Phase 2: 기능 확장 - UI 개인화

MVP 기능 위에 로고, GNB 메뉴 등 UI 브랜딩 기능을 추가하여 완전한 '전용몰' 경험을 제공합니다.

STEP 1: 관리자 기능 개발 (Admin)

로고 관리: 유통사가 전용 로고 이미지를 업로드하는 기능 구현
GNB 관리: 유통사가 GNB 메뉴의 '명칭'과 '링크'를 설정하는 기능 구현

STEP 2: 동적 UI 적용 (Back-end & Front-end)

상품 API 응답에 `logoUrl`, `gnbInfo` 등 추가 데이터를 포함하도록 백엔드 수정
프론트엔드에서 API 데이터를 받아 헤더의 로고와 GNB를 동적으로 변경

4. 주요 사용 시나리오 및 결과

Phase 2까지 완료되었을 때, 각 조건에 따라 사용자에게 보여질 최종 UI 결과 예시입니다.

5. 장기 로드맵 (Long-term Roadmap)

커스텀 도메인 도입
